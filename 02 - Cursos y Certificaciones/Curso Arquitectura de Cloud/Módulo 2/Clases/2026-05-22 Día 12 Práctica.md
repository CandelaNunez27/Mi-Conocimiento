### Pequeña muestra de como se utiliza DockerHub

1. Descargar docker  con DockerHub:
	Vamos a https://hub.docker.com/ y buscamos hola-mundo https://hub.docker.com/_/hola-mundo
	
	 Le damos a los 3 puntos de al lado del botón "Run in Docker Desktop" y copiamos el comando que nos aparezca `sudo docker pull hola-mundo`.

# Práctica

# Despliegue de arquitectura con docker y balanceador. Para simular una página con demanda.

### Preparación

1. Necesitamos ciertas carpetas y archivos:
	Una carpeta llamada app que contendrá el Dockerfile (un texto plano con recetas de sentencias) y un index.php (un archivo de programación para paginas web)
	
	Una carpeta llamada nginx que contiene un archivo nginx.conf (es un archivo se configuracion del servicio web, parecido a apache, que esta funcionando como ALB Application Load Balancer que distribuye el tráfico hacia el servicio web-app de Docker)
	
	Un archivo compose.yaml (será el guiador del docker, con dos bloques de código, los servicios del balanceador y los servicios de web-app)

2. Arrancamos el docker:
	Nos hubicamos en la carpeta que contiene todo y tiramos `sudo docker compose up -d --build` 
	![](../../../../04%20-%20Otros/Imagenes/Pasted%20image%2020260607033318.png)
	
	Si ocurriera un error se puede tirar `sudo docker logs "4 numemos de container ID"`
	
	Podemos verlo en el navegador con `localhost:8081`
	![](../../../../04%20-%20Otros/Imagenes/Pasted%20image%2020260607034353.png)

3. Nos preparamos para la alta disponibilidad:
	Tiramos `sudo docker compose up -d --scale web-app-3` que nos creara 3 servidores más como lo hace el ASG Auto Scaling Group
