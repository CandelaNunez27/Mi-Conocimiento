# Crear maquina con consola propia

### Preparación
1. Clonamos el repo del profesor https://github.com/juliangigena/course_ing_cloud.git en visual studio code.
2. Se debe tener instalado aws; en mi caso use `sudo pacman -S aws-cli-v2`
3. Se puede verificar el .yaml antes de ejecutarlo con `aws cloudformation validate-template --template-body file://infra.yaml`
![](../../../../04%20-%20Otros/Imagenes/Pasted%20image%2020260603034351.png)

4. Se tiran los siguientes comandos
	- `aws configure`
	![](../../../../04%20-%20Otros/Imagenes/Pasted%20image%2020260603030923.png)
	
	- Nos movemos a la carpeta que contiene el .yaml
	![](../../../../04%20-%20Otros/Imagenes/Pasted%20image%2020260603031150.png)

### Ejecución del yaml
1. Ejecutamos la estructura 
```
	  aws cloudformation create-stack \
      --stack-name Lab-FormaTEC-Infra \
      --template-body file://infra.yaml \
      --capabilities CAPABILITY_IAM
```
![](../../../../04%20-%20Otros/Imagenes/Pasted%20image%2020260603032618.png)
	Este comando le indica el nombre con stack-name, se le indica la receta de la estructura esta en esa dirección de archivo, y capabilities es para permitirle los permisos para que trabaje tranquilo.

2. Tardara unos 10 min en crearse todo (se ve el log en la intefaz web > CloudFormation > Event History)

### Conectarnos con la Instancia creada
1. Crear el rol que nos permina conecta
2. Colocarle la base de datos a EC2
	Ir a EC2 > Instancias > y le damos a conectar. 
	![](../../../../04%20-%20Otros/Imagenes/Pasted%20image%2020260603040418.png)
	Le vamos a tener que modificar el rol de IAM > le colocamos este AmazonSSMManagedInstanceCore
	Donde se nos abre su terminal y tiramos `sudo dnf install mariadb105 -y`
- Abrir la maquina creada y Descargamos base de datos mariaDB
- en aws > rds > vemos que tenemos la base de datos y nos da los comandos para conectarlo con la maquina
